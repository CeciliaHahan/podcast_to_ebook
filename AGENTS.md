# Project Rules

These are the default collaboration rules for this repository.

1. First think through the problem, read the codebase for relevant files, and write a plan to `tasks/todo.md`.
2. The plan should have a list of todo items that can be checked off as work is completed.
3. Before beginning implementation, check in with the user and wait for plan verification.
4. Then begin working on the todo items, marking them complete as progress is made.
5. At every step, provide a high-level explanation of changes made.
6. Keep every task and code change as simple as possible; avoid massive or complex changes and minimize code impact.
7. Add a `Review` section to `tasks/todo.md` with a summary of changes and other relevant notes.
